jobs — every scraped job. Key columns: job_id, title, company, url, source, date_posted, llm_score, llm_summary, salary_*, cover_letter_draft, status (new/applied/dismissed/saved). Primary read/write table for the whole pipeline.
bot_state — key/value store. Holds scraper settings (setting_*), last-run timestamps, cv_data:{uid}, telegram offset. PUBLIC readable via anon key — never store secrets here.
user_preferences — per-user settings: keywords, location, alert frequency, Telegram chat_id, min score threshold. Read by user_alerts.py to match jobs per person.
profiles — Auth user profile: email, display name, role (admin/user), created_at. Linked to Supabase auth.users.
user_alert_log — dedup log. PK = (user_id, job_id, channel). Insert only via service_role. Prevents duplicate Telegram/email alerts per user per job.
user_skill_gaps — per-user AI skill gap analysis results. Populated by enricher or web API. Shown on job detail modal.
telegram_claude_history — conversation history for the "Talk to Me" / Claude chat feature per Telegram user. Rolling context for follow-up questions about jobs.

job-alert.yml — Main pipeline. Runs every 5 min on ubuntu-22.04 (pinned: 24.04 breaks Playwright libasound2). Steps: worker.py → user_alerts.py --mode instant. GitHub throttles to ~1 run/10min in practice; cron-job.org pinger is the fix.
enricher.yml — Runs enricher.py: fetches descriptions for unscored jobs, calls Groq llama-3.3-70b-versatile, writes llm_score + llm_summary + cover_letter_draft back to Supabase. Also fires from Windows worker after new jobs land.
daily-digest.yml — Runs digest.py --hours 24 --top 3. Sends daily email summary of top-scored new jobs. Uses Resend API.
user-alerts.yml — Runs user_alerts.py with mode = instant or digest depending on time. Supports multi-user: each user gets alerts for their own keyword/location preferences.
health-check.yml — Runs health_check.py. Checks if worker last ran within expected window; sends Telegram alert if pipeline is down.
easy-apply.yml — Experimental auto-apply via Playwright. Currently paused. apply_executor.py handles the actual form submission logic.
build-apk.yml — Builds Flutter Android APK. Mobile app exists in /mobile but is not the primary interface.

worker.py — Main orchestrator. Reads config from env/bot_state, calls each scraper, pipes results through relevance_engine, deduplicates via dedup.py, syncs to Supabase via db.sync_jobs(), sends Telegram alerts. Handles Telegram callbacks (👍/👎, cover_, cv_ buttons). Also polls callback_query and sets job status in Supabase.
enricher.py — Offline LLM enricher. Reads jobs with llm_score IS NULL, fetches full description, calls Groq llama-3.3-70b-versatile, writes score + summary + cover_letter_draft. Fires from Windows GUI after scan. NOT called by worker.py — separate workflow.
user_alerts.py — Multi-user alert sender. Queries user_preferences, matches jobs, deduplicates against user_alert_log, sends Telegram or email. Supports instant and daily digest modes.
relevance_engine.py — 5-tier classifier:
T5: hard-reject hospitality cluster (housekeeping, concierge, front desk…)
T1P: rescue help desk / service desk phrases
POS: phrase subset match against 64 built-in IT roles + CV titles + keywords
T_DESC: fallback description keyword check
_phrase_token_set() normalises "Information Technology" → "it".
db.py — Supabase helpers. sync_jobs(): upserts jobs, blocks domains (_BLOCKED_DOMAINS: jobsora.com), returns summary dict. set_job_status(): updates status for 👍/👎 feedback. get_bot_state() / set_bot_state() for config reads/writes.
linkedin.py — Scrapes LinkedIn Jobs with li_at cookie. Uses requests + BeautifulSoup. Cloud: skips when datacenter IP detected. Local Windows worker: runs with residential IP.
indeed.py — Scrapes Indeed. Blocked on GitHub Actions (datacenter IP). Only runs from Windows worker.
adzuna.py — Adzuna Jobs API. Requires ADZUNA_APP_ID + ADZUNA_APP_KEY secrets. Runs in cloud and locally.
bayt.py — Scrapes Bayt.com (Gulf job board). HTTP scraping.
gulftalent.py — Scrapes GulfTalent.com. HTTP scraping.
naukri_gulf.py — Scrapes NaukriGulf.com. HTTP scraping.
websearch.py — Tavily web search fallback for job discovery. Uses TAVILY_API_KEY.
gmail_scan.py — Scans Gmail inbox for job alert emails and parses them into jobs. Uses Gmail App Password.
telegram_notify.py — Sends Telegram messages. send_job_alert_with_button(): job card with Apply, Cover Letter, Analyze buttons + 👍/👎 row. answer_callback_query(): popup confirmation.
email_notify.py — Sends email via Resend API. Used by user_alerts.py for non-Telegram users.
digest.py — Daily digest: fetches top-N jobs from last 24h, formats and emails them.
preferences.py — Reads user applied/dismissed history from Supabase, builds few-shot examples for Groq scoring prompt. This is the "learning loop" — liked jobs improve future scoring.
salary.py — Extracts salary from job description via regex + Groq. Writes salary_min/max/avg/currency/period/salary_source. salary_source = "ai_estimate" → displayed as "~ X est." in UI.
dedup.py — Deduplicates jobs across sources by URL and title+company similarity before inserting to Supabase.
dashboard.py — Local monitoring web server (localhost only). Shows pipeline status, recent GitHub Actions runs, Supabase stats, live log tail. DNS-rebinding protected (Host-header validation). Launch via Open-Dashboard.bat.
crew_worker.py — Experimental CrewAI multi-agent rewrite (NOT in production). 6 agents: 4 scouts (LinkedIn/Bayt/GulfTalent/NaukriGulf) + Analyst (Groq scoring) + Alert Dispatcher. worker.py is the active pipeline.

/app/feed — Main job list. Fetches from Supabase jobs table. Supports status filter (new/applied/dismissed/saved), search, sort by score/date. "View as" param for admin impersonation. Each card shows: score badge, salary (with est. label), Analyze, Cover Letter, Talk to Me buttons.
/app/admin — Admin-only (checks user.email === ADMIN_EMAIL). 3 tabs: Scraper Settings (bot_state key-value editor, writable keys allowlist), Users (list + impersonate via "View as"), Recommended Settings (AI-suggested config).
/app/settings — User preferences: keywords, location, alert frequency, Telegram chat_id, min score. Also: CV upload trigger, test Telegram message button (needs TELEGRAM_BOT_TOKEN in Vercel env vars).
/app/saved — Saved jobs (status = 'saved'). Same card layout as feed.
/onboarding — First-run setup: upload CV, set location and keywords, connect Telegram. Shown after first login before feed.
/ (landing) — Marketing landing page with Login / Sign Up CTAs.
JobCard — Main job card component. Shows: title, company, location, date, score badge, salary line (~ prefix + "est." for AI estimates). Actions: Apply (URL), Analyze (JobDetailModal), Cover Letter (CoverLetterModal), Talk to Me (ChatWidget).
JobDetailModal — Full job detail overlay. Shows description, llm_summary, matched_skills, missing_skills, red_flags, salary breakdown (with est. labeling).
CoverLetterModal — Auto-generates on open via POST /api/app/cover-letter/[id]. Editable textarea, Copy (with ✓ Copied feedback), Regenerate, Close. Escape key closes. Requires GROQ_API_KEY in Vercel env vars.
ChatWidget — "Talk to Me" chat about a specific job. Calls /api/app/chat. Maintains conversation history in telegram_claude_history table.
CVUploadCard — CV upload UI. Calls /api/app/cv/upload. Parsed CV stored as cv_data:{uid} in bot_state (fields: summary, years_experience, job_titles[], skills[], certifications[]).
GET /api/app/jobs/[id] — Fetches single job detail. Uses explicit JobDetailRow interface + .single<JobDetailRow>() to avoid TypeScript GenericStringError (this was the Vercel build-blocking bug, fixed in commit 3de061b).
POST /api/app/cover-letter/[id] — Fetches job + user CV from bot_state, builds system prompt, calls Groq llama-3.3-70b-versatile, returns 180-250 word plain-text cover letter. Requires GROQ_API_KEY in Vercel env vars.
cv/upload + cv/profile — Upload and parse CV. Writes structured JSON to bot_state key cv_data:{uid}.
POST /api/app/chat — LLM chat about a job. Uses Groq. Stores history in telegram_claude_history table.
admin/users + admin/bot-state — Admin API routes using service_role client. bot-state writes restricted to _WRITABLE_SETTINGS allowlist only.
POST /api/telegram/webhook — Receives Telegram webhook events (alternative to worker.py polling). Not yet active as primary handler — worker.py still polls.
GET /api/cron/trigger-worker — Vercel cron endpoint to trigger GitHub Actions workflow_dispatch. Backup for GitHub's 5-min throttling issue.

linkedin-job-worker.ps1 — Background worker loop. Every 5 min: calls worker.py (cloud pipeline) via Start-Job, then fires enricher.py async if new jobs found. Polls Telegram callbacks. Manages worker.pid, seen-jobs.json, worker.log.
linkedin-job-alert.ps1 — System tray GUI. Shows lamp indicators (Local 🟢/🔴, Cloud 🟡/⚫), last scan time, job count. Buttons: Run Now, Open Dashboard, View Logs, Settings. Launched hidden by Start-LinkedInJobWorker-Hidden.vbs.
job-database.ps1 — Local Supabase helpers in PowerShell. Reads/writes settings, job records, seen-jobs list. Dot-sourced by both worker and alert scripts.
shared-functions.ps1 — Shared utilities: HTTP client helpers, Telegram send functions, log formatters. Dot-sourced by worker and alert scripts.
settings.json — Local config file (not in git). Contains SupabaseURL, SupabaseKey (currently anon key — pending: replace with service_role key), TelegramToken, TelegramChatId, etc.
Open-Dashboard.bat — Opens localhost dashboard (dashboard.py must be running first). Shows pipeline status, recent runs, Supabase stats, live logs.
Install-LinkedInJobWorkerTask.ps1 — Registers a Windows Task Scheduler task to auto-start the worker on login. Run once during setup.
